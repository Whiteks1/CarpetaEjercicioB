Menú Emergente

Este menú tiene sus elementos anidados no visibles hasta que hacemos hover sobre él


.botonera {
    display: none;
}
/*modifico el elemento directamente*/
.mi-btn-menu:hover {
    opacity: .5;
}
/*modifico un elemento anidado del botón que recibe el hover*/
.mi-btn-menu:hover .botonera {
   display: block;
}

Anidación directa e indirecta con CSS

Anidación indirecta

Un selector en CSS separado de otro por un espacio significa que el formato afectará a cualquier elemento del segundo selector que esté dentro del primero, con independencia del nivel de anidación entre ellos.

En este ejemplo todos los elementos li (tanto de ul como de ol) se modifican con las reglas CSS definidas.

    color: darkred;
    border:1px solid;

}
.disparador-a:hover li {
    background-color: darkred;
    color: white;
    padding-left: 2rem;
}

Anidación Directa

Si solamente queremos acceder a un elemento que esté directamente anidado en otro usaremos el operador > entre los dos selectores.

En el siguiente ejemplo eliminamos las viñetas de los elementos li que están dentro del ul disparador-b, pero no los de los li del ol que está también anidado dentro de disparador-b.

Selectores Adyacentes

Menú Emergente no Anidado

Los selectores adyacentes no son muy útiles con el evento hover, puesto que el menú que mostramos al hacer hover sobre el botón desaparece cuando intentamos llegar a él (porque el botón disparador pierde el hover). Son más útiles para comportamientos de tipo tooltip.

Este cuadro funciona como un tooltip

En este caso es necesario aplicar un position absolute al elemento emergente para que no desplace el contenido visible en la pantalla.

Evento checked junto a selector adyacente

Para mostrar y mantener visible un elemento oculto con CSS usamos un input de tipo checkbox y el selector adyacente. Cuando activamos el input usamos su evento checked para hacer visible en menú que se encuentra a continuación del input en el DOM.


.menu-oculto li {
    list-style: none;
    margin: .5rem 0;
}
.menu-oculto, .mns-oculto {
    display: none; 
    position: absolute;
    background-color: white;
}
.dp-adyacente:hover  + .mns-oculto {
    display: block;
}
#sel-menu:checked + .menu-oculto {
    display: block;
}

Menú emergente animado

En este ejemplo vamos a combinar los selectores de CSS adyacentes con las transiciones para crear un menú desplazable que aparezca en pantalla de forma animada.

A tener en cuenta

  • Necesitamos trabajar con position relative y position absolute para poder ubicar bien los elmentos y para definir el desplazamiento.
  • Necesitamos un elemento contenedor al que aplicar overflow hidden para mantener oculto el menú
  • Usaremos un input checkbox para cambiar el estado de visibilidad del menú

.pantalla {
    overflow: hidden;
    min-height: 350px;
    position: relative;
}
.dpl-grupo {
    width: 180px;
    left: -210px;
    position: absolute;
    transition: all .2s;
    background-color: white;
}
#btn-dpl:checked + .dpl-grupo {
    left: 0;
}
.dpl-content {
    transition: all .2s;
    margin-top: 0;
}
#btn-dpl:checked + * + .dpl-content {
    margin-top: 220px;
}
